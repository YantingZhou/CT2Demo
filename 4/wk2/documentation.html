My "Rube Goldberge Machine" links to 4 APIs. Each of them is somehow related to my cat -- Toto.

The NASA API Astronomy Picture of the Day was fetched according to the date when Toto is born.

The second API Petfinder was to find the nearest cat lady near Toto. It uses OAuth for secure authentication. The instruction from their website uses curl. I used a online code translator convert it to javascript.

I created 3 constant for fixed url, key, and secret, and a variable for token which is going to change each time fetched.

Because it is a 2 step process (authentication and fetch API), I separate one function into two, each of them contains a fetch method. I'm not sure if this is the best way to fetch data, but at least it worked.

Data! The result is from the URL, which contain query of female and the zip code of my home

The only problem is: if the new listing of pet doesn't have a photo, the photo array will be empty, causing error.

This is how the section shows if it functions well.
